✅ PART 1 — 10 Most-Asked Backend Interview Questions

These come up again and again for Spring Boot / microservices roles.

⭐ 1) What happens when you type a URL in the browser?

They expect:

DNS lookup

request hits load balancer

reaches server

controller handles request

DB call

response returned

👉 Tests end-to-end understanding.

⭐ 2) How do microservices communicate?

Good answer:

REST (HTTP)

Messaging (Kafka/RabbitMQ)

Feign client for internal calls

Bonus:

Sync vs Async communication tradeoffs.

⭐ 3) What is @Transactional?

Explain:

ensures atomic DB operations

rollback on failure

maintains consistency

Bonus:
Isolation levels.

⭐ 4) How do you handle failures in microservices?

Say:

retries

circuit breaker (Resilience4j)

fallbacks

proper logging

Interviewers LOVE this.

⭐ 5) Stateless vs Stateful services?

Stateless:

no session stored

scalable

preferred for microservices

Stateful:

stores session

harder to scale

⭐ 6) How does caching help?

Explain:

reduces DB load

faster responses

Redis commonly used

Bonus:
Cache invalidation strategies.

⭐ 7) What is DB indexing?

Good answer:

speeds up search

works like book index

tradeoff = slower writes

⭐ 8) How do you secure APIs?

Say:

JWT auth

OAuth

role-based access

HTTPS

⭐ 9) What is idempotency?

Example:

Calling same API twice gives same result


Important for:

payments

order APIs

⭐ 10) How would you design an order system?

Mention:

order service

inventory service

payment service

event-driven flow

DB transactions

👉 This connects directly to your project.
