4️⃣ JWT Authentication Flow

Authentication service generates JWT tokens.

Login request:

POST /api/auth/login

Response:

JWT Token

Gateway validates JWT using custom filter.

JWT Filter Logic
if (path.startsWith("/api/auth")) {
    return chain.filter(exchange);
}

Meaning:

/api/auth/** → public

All other APIs require:

Authorization: Bearer <token>

If missing:

401 Unauthorized
5️⃣ Major Debugging Issues We Solved

These were the main problems fixed during these days.

Problem 1 — 401 Unauthorized from Gateway

Issue:

Spring Security blocked requests before JWT filter ran

Fix:

SecurityConfig → permitAll()
JWT filter handles authentication

Final flow:

Spring Security
      ↓
Allow request
      ↓
JWT Gateway Filter
      ↓
Validate token
Problem 2 — Feign Service Discovery Failure

Error:

Load balancer does not contain an instance for service

Cause:

Service not registered with Eureka

Fix:

spring.application.name configured
Eureka client dependency added
Problem 3 — Cart Service Endpoint Mismatch

Error:

404 /cart/1

Cause:

Feign client used wrong path.

Wrong:

/cart/{id}

Correct:

/api/cart/{id}
Problem 4 — Inventory Service Unique Constraint

Error:

Duplicate entry '1'

Cause:

Database unique constraint on product_id.

Fix:

Update quantity instead of inserting duplicate product

7️⃣ Circuit Breaker (Resilience4j)

Used to protect against service failures.

Annotation used:

@CircuitBreaker(name = "inventoryBreaker", fallbackMethod = "inventoryFallback")

Fallback example:

public Order inventoryFallback(Long userId, Throwable throwable) {
    throw new RuntimeException("Inventory service unavailable");
}

Purpose:

Prevent cascading failure
Provide fallback response

1️⃣4️⃣ Interview Questions (Very Important)

These are exact questions interviewers ask for projects like this.

Architecture

Q: Why do we use API Gateway?

Answer:

API Gateway acts as a single entry point for all client requests.
It handles authentication, routing, rate limiting, and monitoring.

Q: Why use Eureka Service Discovery?

Answer:

Microservices run on dynamic ports.
Eureka allows services to discover each other without hardcoding URLs.

Q: What is Feign Client?

Answer:

Feign simplifies REST API calls between microservices by using declarative interfaces.

Q: What problem does Circuit Breaker solve?

Answer:

Prevents cascading failures when a service becomes unavailable.
It stops repeated failing calls and triggers fallback logic.

Q: Why use Kafka in microservices?

Answer:

Kafka enables asynchronous communication between services.
It decouples producers and consumers and improves scalability.

Q: Why Dockerize microservices?

Answer:

Docker ensures consistent environments across development and production.
It simplifies deployment and scaling.

Q: Difference between synchronous vs event-driven communication?

Synchronous:

Service A calls Service B directly.

Event-Driven:

Service A publishes event → Kafka → Service B consumes.

Benefits:

Loose coupling
Better scalability
Fault tolerance
